4:解压并整理软件
回到虚拟机中来，可通过ls命令查看上传过来的文件

linux软件不需要像windows那样点击exe配置和安装，Linux多数软件只需要解压即可使用（tar tar.gz tar.bz等类型）

对java、storm、zookeeper解压操作；将安装包存放到/root/soft目录下,便于以后寻找或再次安装
解压后分别放置到：
java —— /root/java
zookeeper —— /root/zookeeper
storm —— /root/storm

具体命令如下：

对于解压命令不熟的童鞋可以，通过＃ man tar  命令来查看 tar的用法
 # cd ~
＃tar -zxvf jdk8u74-linux-x64.tar.gz
＃tar -xvf apache-storm-0.10.0.tar
＃tar -zxvf zookeeper-3.5.1-alpha.tar.gz
创建文件夹
对于mkdir命令不熟的童鞋可以，通过＃ man mkdir 命令来查看 mkdir的用法
 # mkdir soft
 # mkdir java
 # mkdir zookeeper
 # mkdir storm

移动文件重命名
对于mv命令不熟的童鞋可以，通过＃ man mv 命令来查看 mv的用法
 # mv jdk8u74-linux-x64.tar.gz soft/
 # mv apache-storm-0.10.0.tar soft/
 # mv zookeeper-3.5.1-alpha.tar.gz soft/
 # mv jdk1.8.0_74/ java/
# mv apache-storm-0.10.0/ storm/
# mv zookeeper-3.5.1-alpha/ zookeeper/
 

5:配置java环境变量，配置zookeeper和storm的环境变量
# vi /etc/profile
按键盘 i 键进入insert模式
在文件末尾追加 如下信息

Linux 下环境变量通常都是
定义环境变量
key=value
声明环境变量
export key

也可同时定义并声明
export key=value

相互引用使用$key(windows使用%key%)
变量间隔使用:(windows使用;)

推出insert模式
ESC
保存并退出
:wq

使配置的环境变量生效
# source /etc/profile
查看java版本信息，测试java是否好用
# java -version

理论上不用配置storm和zookeeper环境变量也可
